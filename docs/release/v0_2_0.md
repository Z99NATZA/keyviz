# v0.2.0 - English And Thai Keyboard Input

Release date: 2026-07-16

KeyViz v0.2.0 adds Unicode input that follows the active Windows keyboard
layout. English and Thai text now appear correctly in the overlay while typing
in applications such as Notepad, Chrome, and VS Code.

## Highlights

- Added English US, Thai Kedmanee, and Thai Pattachote input.
- Added live keyboard-layout switching through `Win + Space` and `Alt + Shift`.
- Added support for hosted text controls used by modern Notepad and other
  applications where the editor and top-level window use different threads.
- Added Unicode-aware history, trimming, and Backspace behavior.
- Added Thai-capable font fallbacks for vowels and tone marks.
- Added automated coverage for English and Thai Windows layout translation.

## Input And Display

Printable text follows the keyboard layout selected in the application where
the user is typing:

```text
hello ที่ Ctrl + S next
```

- Thai letters, vowels, and tone marks render together correctly; `ท`, `ี`, and
  `่` appear as `ที่`.
- Space appears as an ordinary blank character.
- Special keys and shortcuts remain inline green text.
- Backspace removes one Unicode code point at a time, so the text portion
  changes through `ที่` → `ที` → `ท` → empty.
- Text and special labels share the same `maxHistoryLength` limit. `ที่` counts
  as 3, `Shift` as 5, and `Ctrl + S` as 8.

## Settings

The current settings format is:

```json
{
  "maxHistoryLength": 20,
  "showControls": false,
  "bubblePosition": "center"
}
```

For a v0.1.0 settings file, replace `maxTextLength` and `maxSpecialKeys` with
`maxHistoryLength`. Existing settings files are not rewritten automatically;
when `maxHistoryLength` is absent, KeyViz uses the default value of 20.

## Known Limitations

- Dead-key composition is not supported.
- IME composition for languages such as Chinese, Japanese, and Korean is not
  supported.
- Custom editors that expose neither a Win32 focus window nor a caret window may
  fall back to the top-level application's layout.
- Secure Desktop surfaces such as UAC prompts are not captured or overlaid.
- Exclusive-fullscreen games may appear above the overlay.

## Upgrade

Exit every running KeyViz instance before starting v0.2.0 so the Raw Input
listener and System Tray icon are replaced by the new version.
